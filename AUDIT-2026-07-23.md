# Project Audit — 2026-07-23

Full-project review at Wes's request ("nothing is off the table as far as redesign").
Six parallel deep-dive passes: run-mode UX/layout, authoring UX, sound library,
notes access, rules/character correctness, multiplayer + cleanup — plus synthesis.
**This supersedes AUDIT-2026-07-10.md as the working punch list** (the few open
items from that file are folded in below; the old file can be archived).

Overall verdict: the foundation is still healthy and the feature surface is huge
now. The problems cluster in five themes:

1. **Reading and running a scene is harder than it should be** — typography and
   layout never got a design pass, and the bottom console eats the screen.
2. **Authoring has two disconnected mental models** (palette-add vs cue-place)
   and the editor can't do things the docs teach.
3. **The 2,169-asset library is organized by filename accidents**, not by what a
   DM reaches for ("tense", "calm", "battle-but-not-boss").
4. **Notes navigation evicts your context** — one panel, everything fights over it.
5. **The player portal trusts the client** — fine on a couch LAN, not fine the
   moment real players (or a tunnel) show up.

Character math is accurate to 2024 with ONE real bug; the bigger character gap is
that level-up is a reminder machine, not a chooser.

## P2 — Library reorganization (the 2,169-asset problem)

Real numbers (live Elor library): 1,792 sfx / 261 music / 116 ambience.
**65% is two filename buckets** (footsteps ×751, voices ×654). Top "tags" are
import garbage (`wav` 1,773, `mono` 913, pack names). Only 7% of assets carry
any mood word. Only 38 assets use multi-category. 233 near-duplicate variation
families cover 1,593 assets. Nothing records whether a sound was ever
auditioned. Search is unranked substring over a haystack that includes the junk
tags (`LibraryPanel.tsx:95-108`).

- [ ] **Junk-tag cleanup pass** (scripted, one shot): strip `wav/mono/nox/
  essentials/pack/...` tokens, then re-derive tags. Instantly improves every
  search AND every Claude scene-authoring session (which greps these tags).
- [ ] **`audition` status on LibraryAsset** (`unheard | ok | starred`) — mirrors
  the existing `trash?` pattern. Default everything from waves 4-8 to `unheard`;
  filter chip in the browser; Triage stamps `ok`. Finally separates "the 300
  sounds I trust" from the pile.
- [ ] **Mood/energy as a real axis**: add `mood?: string[]` (calm/tense/epic/
  eerie/sad/festive/…) + optional `energy?: 1-3`. This is the axis a DM reaches
  for live ("tense but not combat") and what the combat-music complaint
  (TODOS: everything skews "super boss") actually needs. Backfill by script
  from names/categories where safe; the rest via the browser.
- [ ] **Collapse variation families**: group `footsteps grass walk 01-50` under
  one browsable parent with a takes-count (the strip-trailing-numbers heuristic
  already identifies 233 groups). The footsteps/voices megabuckets become ~40
  rows instead of 1,405.
- [ ] **Bulk edit**: multi-select rows → set category/tags/mood/audition in one
  write (today `updateLibraryAsset` is one-at-a-time). Also expose license/
  source in the ✎ editor — 124 assets say "UNKNOWN — do not redistribute" and
  can only be fixed in raw JSON today.
- [ ] **Ranked search**: use the existing `fuzzyScore` (`lib/fuzzy.ts`) with
  field weighting (name > mood > category > tags) instead of `includes()` +
  alphabetical. Promote favorites into library.json (renderer-local today, so
  Claude authoring can't see them).
- [ ] Virtualize the list (react-window) — expanding a 751-item group renders
  700 live rows today (`GROUP_CAP` expander, `LibraryPanel.tsx:281-288`).

## P3 — Players on their machines (Ember E3 roadmap)

Charter holds (SURFACES-PLAN, locked): **players get the browser app, never the
Electron app** — the desktop app is structurally single-DM (one campaign folder,
full fs write, audio engine). "Players using this on their machines" = Ember in
their browser, reachable off-LAN. What that takes, in order:

1. P0 security items above (server-side fog filter, ownership tokens, passphrase).
2. **Remote route — Wes decides**: Tailscale (zero config after install, LAN-ish
   security model) vs Cloudflare Tunnel (real URL, TLS, no installs for players)
   vs port-forward (free, riskiest). Recommendation: **Cloudflare Tunnel** —
   players just open a link on any device, TLS included.
3. **Handout/image push to the portal** — the biggest missing player feature;
   DM-side push exists but only reaches the presenter window over IPC
   (`index.ts:417,498`); the portal SSE has no image event. Add `/api/handout` +
   SSE `image`, with a player-side lightbox.
4. **"It's your turn" nudge** — initiative order already flows to the table view;
   add an active-turn highlight + optional notification.
5. Later, as the table demands: chat/whispers, shared player-visible notes,
   browser audio (big; Discord already covers sound).
