# Project Audit — 2026-07-23

Full-project review at Wes's request ("nothing is off the table as far as redesign").
Six parallel deep-dive passes: run-mode UX/layout, authoring UX, sound library,
notes access, rules/character correctness, multiplayer + cleanup — plus synthesis.
**This supersedes AUDIT-2026-07-10.md as the working punch list** (the few open
items from that file are folded in below; the old file can be archived).

Overall verdict: the foundation is still healthy and the feature surface is huge
now. The problems cluster in five themes:

1. **Reading and running a scene is harder than it should be** — typography and
   layout never got a design pass, and the bottom console eats the screen.
2. **Authoring has two disconnected mental models** (palette-add vs cue-place)
   and the editor can't do things the docs teach.
3. **The 2,169-asset library is organized by filename accidents**, not by what a
   DM reaches for ("tense", "calm", "battle-but-not-boss").
4. **Notes navigation evicts your context** — one panel, everything fights over it.
5. **The player portal trusts the client** — fine on a couch LAN, not fine the
   moment real players (or a tunnel) show up.

Character math is accurate to 2024 with ONE real bug; the bigger character gap is
that level-up is a reminder machine, not a chooser.

## P0 — Trust fixes (before players touch it, and one rules bug)

### Portal security (must close before any remote play, worth closing on LAN too)
- [ ] **The full live map ships to player browsers** — hidden monster tokens,
  fog-zone geometry, and the un-fogged map image are all in `GET /api/table`;
  the DM-secret filtering happens *client-side* (`playerTableView` runs in the
  player's browser, `MapEditor.tsx:627-654`; raw map returned at
  `index.ts:464-467`). Any player can curl the endpoint and see everything fog
  hides. Move the filtering server-side.
- [ ] **Any browser can overwrite any character.** `POST /api/character/:id`
  (`playerServer.ts:159-168`) never checks ownership — identity is just
  localStorage. Same self-asserted `who` on `/api/roll` and `/api/party`
  transfers. Minimum fix: per-player claim token issued when a character is
  first picked (stored server-side), checked on every mutation.
- [ ] **No auth at all** (`playerServer.ts:13`, binds 0.0.0.0). Add a campaign
  passphrase (URL token is fine) — required before any tunnel/remote route.
- [ ] No rate limiting / origin checks on mutating POSTs; CORS wide open
  (`index.ts:177`). Cheap hardening once auth exists.

### Rules bug — RETRACTED after verification
- [x] ~~Multiclass half-caster spell slots round the wrong way~~ — the review
  agent applied the **2014** rule (round down). The 2024 PHB multiclass rule
  rounds paladin/ranger levels **up**, which is exactly what `character.ts:121`
  does. Code is correct; no change. (All 14 hand-coded-table spot-checks pass.)

## P1 — The run surface (readability + the bottom console)

This is the design pass Wes asked for. Approach per the playbook: **build a
proof-of-concept of the redesigned run screen first, screenshot it, get his
reaction before rolling it everywhere.**

### Script readability (all in `ScriptPanel.tsx` / `ControlBoard.tsx`)
- [ ] **No line-length cap** — run mode renders prose full-window-width (120+
  chars/line). Add `max-w-[68ch] mx-auto` at the `main`/script level. The single
  biggest read-aloud fix.
- [ ] **`leading-loose` (2.0) at 18px is over-spaced** (`ScriptPanel.tsx:192`) —
  drop to ~1.6; consider bumping body to 19-20px in run mode (it's a teleprompter).
- [ ] **Heading hierarchy inverts**: h1/h2 differ by one step; h3 is *dimmer than
  body text* (`HEADING_CLASS`, `ScriptPanel.tsx:22-26`). Sections drive ambience
  lifecycle — make them landmarks (size + ember rule + whitespace).
- [ ] **DM callouts are under-marked in the read view**: the 🕯 DM label exists
  only in editor CSS (`index.css:81-90`), not in `renderBlock`
  (`ScriptPanel.tsx:311-319`). The one thing that must never be read aloud is
  the least distinct thing on the page. Port the label + stronger tint.
- [ ] **Cue chips are smaller than the prose they sit in** (`text-sm` inside 18px
  text) — actionable things should be ≥ body size with a clearer fire affordance;
  the teleprompter "next" ring is a 1px whisper.
- [ ] Teleprompter key hint is 10px at 70% opacity (`ScriptPanel.tsx:167`) —
  the primary run interaction is invisible. Fold into the `?` cheat-sheet and
  make the on-screen hint legible.
- [ ] ~4 chrome elements stack before the first word of prose (Read-aloud header,
  h2 scene title, divider, dmNotes) — collapse into one compact scene header.

### The bottom console (`SoundConsole.tsx`)
- [ ] **Split "what's sounding" from "what I can fire."** The dock does double
  duty: NOW row (mixer/kill) + three SCENE palette rows + playlists + staples =
  5 wrapping rows hitting the 224px cap and *internally scrolling* mid-session.
  Proposed shape (mock it up first): a slim always-on NOW/transport strip
  (with the progress scrubber that today is stranded in the build-only
  `NowPlayingStrip`, `MusicPalette.tsx:156-235`), and the scene palette as a
  right-panel tab or collapsible section — reclaiming ~150px of script space.
- [ ] **Two look-alike chip types behave differently** (NowChip = fader/kill,
  FireChip = trigger) and a playing track appears in both rows. Distinct visual
  grammar (playing = horizontal bar w/ progress; fireable = button).
- [ ] Row labels still use the retired glyphs `♪/〜/⚡/🎜`
  (`SoundConsole.tsx:554,576,598,621`) — the MUS/AMB/SFX text-badge migration
  stopped at this file. Finish it.
- [ ] Mixing is split across the screen: bus sliders live in the TopBar
  (`TopBar.tsx:215-224`), per-item faders in the dock. Co-locate in the dock's
  mixer; TopBar sheds 4 of its 11 controls (old audit's "TopBar overload").

## P1 — Notes at hand (the "don't take me somewhere else" fix)

Current model: one right-panel slot; every route to a note **evicts** what's
there and resets scroll (link click force-switches the panel tab,
`ControlBoard.tsx:290-292`; a link inside the panel note navigates the panel
away in place). The hover peek is the only glance-without-leaving surface and
it dies on mouse-out (`NotePeek.tsx:49-51`, hover-lifetime only).

- [ ] **Pinnable peek cards.** The peek already portals to body — add a 📌 that
  promotes it to a persistent draggable card (state moves from the per-pill hook
  into the store; allow 2-3 pinned at once). This is the highest-value change:
  glance at Soni's note while the script and cast stay put.
- [ ] **"In this scene" auto-index.** A scene doesn't know which notes it
  references — `docLinks(scene.script)` exists but is never used forward. Derive
  a per-scene entity strip (script links ∪ Cast rows) shown in run mode; hover =
  peek, click = pin. Kills most mid-session Ctrl+K trips.
- [ ] **Stop the eviction**: link clicks from the script should open a peek/pin,
  NOT hijack the right-panel tab; in-panel link clicks should offer peek instead
  of navigating the panel away. (History stack stays for deliberate navigation.)
- [ ] **Recent-notes row** in the 📓 tab (last ~5 viewed) — the back-stack exists
  (`store.ts:1193-1225`) but is invisible.
- [ ] **Findability**: NotesRail has no search box (pure scroll over 91 notes);
  Ctrl+K has no kind filter. Add rail filter + `kind:` scoping; 24 NPCs as one
  flat alphabetical list needs at least favorites-first or arc grouping (tags
  exist on scenes' library, not on notes — add `tags` to CampaignNote).

## P2 — Authoring (text + adding audio)

- [ ] **`{{sfx:...}}` typed in the editor does nothing** — no input rule for
  `CueNode` (`extensions.ts`), yet every scene template *teaches that syntax*
  (`campaign.ts:66-105`), and `scriptText` dies on first in-app save. Add the
  input rule (pattern already exists for `{{monster:}}`, `extensions.ts:177-192`).
- [ ] **No audition before placing** — tray chips and the ⚙ popover play
  nothing; you place sounds blind. `previewAsset`/`PreviewScrubber` already
  exist (`LibraryPanel.tsx:379-388`) — wire ▶ into tray chips + the popover.
- [ ] **One "add sound" flow instead of two.** Palette `+ Add` (modal, no cue)
  vs editor tray (cue + auto-palette side effect) is a conceptual trap. Build a
  `/`-triggered in-editor command palette on the proven `LinkSuggest` engine
  (`LinkSuggest.tsx:39-140`): type `/tense` → fuzzy library results with ▶
  audition → Enter inserts cue + registers to palette. Palette `+ Add` becomes a
  thin inline search (no full-screen modal) using the same component.
- [ ] **⚙ popover can't retarget outside the scene** (`CueChip.tsx:10-22`
  scene-palette-only; orphaned refs are a dead end) — combine with
  `ensureAsset` to allow full-library retarget + self-heal. Also: volume/fade
  controls are amb-only; extend to music (fade) and sfx (volume).
- [ ] **`[[links]]` can't be authored in the read-aloud editor** — schema
  supports them, but `LinkSuggest` is only mounted in `NoteEditor`
  (`NoteEditor.tsx:147`). Wrap `ScriptEditor` too.
- [ ] Bullet/numbered lists are disabled in the editor (`extensions.ts:344-352`)
  — a DM can't make a plain list. Enable bulletList/orderedList (compiler +
  mapping.ts need the block type).
- [ ] mapping.ts silently deletes unknown marks/blocks (`mapping.ts:99,170`) —
  log loudly in dev so future block types can't round-trip to data loss.

## P2 — Library reorganization (the 2,169-asset problem)

Real numbers (live Elor library): 1,792 sfx / 261 music / 116 ambience.
**65% is two filename buckets** (footsteps ×751, voices ×654). Top "tags" are
import garbage (`wav` 1,773, `mono` 913, pack names). Only 7% of assets carry
any mood word. Only 38 assets use multi-category. 233 near-duplicate variation
families cover 1,593 assets. Nothing records whether a sound was ever
auditioned. Search is unranked substring over a haystack that includes the junk
tags (`LibraryPanel.tsx:95-108`).

- [ ] **Junk-tag cleanup pass** (scripted, one shot): strip `wav/mono/nox/
  essentials/pack/...` tokens, then re-derive tags. Instantly improves every
  search AND every Claude scene-authoring session (which greps these tags).
- [ ] **`audition` status on LibraryAsset** (`unheard | ok | starred`) — mirrors
  the existing `trash?` pattern. Default everything from waves 4-8 to `unheard`;
  filter chip in the browser; Triage stamps `ok`. Finally separates "the 300
  sounds I trust" from the pile.
- [ ] **Mood/energy as a real axis**: add `mood?: string[]` (calm/tense/epic/
  eerie/sad/festive/…) + optional `energy?: 1-3`. This is the axis a DM reaches
  for live ("tense but not combat") and what the combat-music complaint
  (TODOS: everything skews "super boss") actually needs. Backfill by script
  from names/categories where safe; the rest via the browser.
- [ ] **Collapse variation families**: group `footsteps grass walk 01-50` under
  one browsable parent with a takes-count (the strip-trailing-numbers heuristic
  already identifies 233 groups). The footsteps/voices megabuckets become ~40
  rows instead of 1,405.
- [ ] **Bulk edit**: multi-select rows → set category/tags/mood/audition in one
  write (today `updateLibraryAsset` is one-at-a-time). Also expose license/
  source in the ✎ editor — 124 assets say "UNKNOWN — do not redistribute" and
  can only be fixed in raw JSON today.
- [ ] **Ranked search**: use the existing `fuzzyScore` (`lib/fuzzy.ts`) with
  field weighting (name > mood > category > tags) instead of `includes()` +
  alphabetical. Promote favorites into library.json (renderer-local today, so
  Claude authoring can't see them).
- [ ] Virtualize the list (react-window) — expanding a 751-item group renders
  700 live rows today (`GROUP_CAP` expander, `LibraryPanel.tsx:281-288`).

## P3 — Players on their machines (Ember E3 roadmap)

Charter holds (SURFACES-PLAN, locked): **players get the browser app, never the
Electron app** — the desktop app is structurally single-DM (one campaign folder,
full fs write, audio engine). "Players using this on their machines" = Ember in
their browser, reachable off-LAN. What that takes, in order:

1. P0 security items above (server-side fog filter, ownership tokens, passphrase).
2. **Remote route — Wes decides**: Tailscale (zero config after install, LAN-ish
   security model) vs Cloudflare Tunnel (real URL, TLS, no installs for players)
   vs port-forward (free, riskiest). Recommendation: **Cloudflare Tunnel** —
   players just open a link on any device, TLS included.
3. **Handout/image push to the portal** — the biggest missing player feature;
   DM-side push exists but only reaches the presenter window over IPC
   (`index.ts:417,498`); the portal SSE has no image event. Add `/api/handout` +
   SSE `image`, with a player-side lightbox.
4. **"It's your turn" nudge** — initiative order already flows to the table view;
   add an active-turn highlight + optional notification.
5. Later, as the table demands: chat/whispers, shared player-visible notes,
   browser audio (big; Discord already covers sound).
