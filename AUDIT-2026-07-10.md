Full-project review (3 parallel audit passes: UX coherence, code health, docs/content)
plus synthesis. **This is the working punch list — check items off as they land and

## P0 — Notes read-first redesign + table bugs ✅ DONE 2026-07-10

### The notes interaction disease
The app's good rule (scene scripts follow it): **read-first; ✎ Edit is a deliberate
step; plain-click links navigate.** The note page breaks it — it is a live TipTap
editor at all times, so:
- [x] **NoteView → read-first.** Render `NoteBody` (read-only, plain-click links,
  hover peeks, live checklists) by default; mount `NoteEditor` only behind a
  ✎ Edit toggle (build mode only) — copy the ScriptPanel pattern verbatim.
  Fixes: click-a-link-gets-formatting-bubble (the DM's complaint), the lying
  `cursor-pointer` on chips, inconsistent link behavior across surfaces.
- [x] **Run-mode guard:** `ControlBoard.tsx` shows NoteView with no `uiMode` check —
  in run mode with leftTab=notes the *editor* replaces the script. Run mode should
  show the read view (or push notes to the right panel).
- [x] **Ideas/Cast add-inputs are live in run mode** while every other authoring
  affordance is build-gated — gate the add/rename inputs (keep checkbox ticking,
  that's session tracking).

### Table bugs (each < 30 min, from the code audit)
- [x] **Preview leaks to Discord**: `AudioEngine.preview()` connects to sfxBus→master,
  and the Discord tap hangs off master. Auditioning a sound in the Library mid-session
  streams it to players. Reconnect preview to the monitor node (post-tap).
- [x] **Non-atomic JSON writes** (`campaign.ts` — 8+ call sites): crash mid-write
  truncates the file; worst case library.json (2,311 assets, rewritten on every tag
  edit). Add `writeJsonAtomic` (tmp + rename), swap all call sites.
- [x] **Campaign switch wipes the Discord token**: `campaign.ts writeConfig` writes
  `{campaignPath}` wholesale over the same hearth-config.json that `discord.ts
  writeToken` merge-writes. Make writeConfig read-modify-write.
- [x] **Stale activeMusic after natural track end**: non-looping palette tracks get no
  `onended`; when one plays out, Now Sounding shows it forever and re-tapping it is a
  silent no-op (`switchMusic` early-return). Add onended → clear activeMusic.
- [x] **updateScene save failures are silent** (`store.ts` catch → console only, while
  updateNote toasts). Copy the toast.
- [x] **Watcher echo-suppression can drop external edits**: any fs event within 1500ms
  of an internal write is discarded with no re-check (`campaign.ts:311`) — Claude's
  file edit during an autosave window is silently lost, then overwritten. Defer a
  reload instead of dropping.
- [x] **Probe memory bomb**: `probeAssets` fires an unbounded Promise.all full-read of
  every referenced file (potentially GBs). Cap concurrency / use an existence check
  IPC instead of full fetches.

## P1 — Discoverability + consistency ✅ DONE 2026-07-10

- [x] **Ctrl+K has zero visible entry point.** Add a search affordance (TopBar or rail
  header) labeled with the shortcut.
- [x] **Shortcut cheat-sheet** (`?` overlay or Tools menu): Ctrl+K, Ctrl+J, Space /

  learn they exist. Always render (disabled) on the note page, or add to the hint.

  as noise live") but script cues/tray/chips still use ♪/🔊/〜. Pick the text badges
  everywhere the DM reads mid-session.
- [x] Hover-only critical actions (scene ✎/⧉/📅/🗑, image ✕, idea/cast ✕/⤴, playlist ✕)
  — consider persistent-but-dim instead of opacity-0.
- [x] `window.confirm` for deletes → in-app confirm (three call sites).
